[Query-1] Contract Query Requests

A. [GET] api/v1/request/query

  • Description
    • API for retrieving stored data in equilibrium network.
    • microChainId, chainNetworkId and chainId are used for specifying user’s network. User’s are able to get detailed information about Network by calling [Network-1-A] API.
  • Request
    • Method : GET
    • Path : api/v1/request/query
    • HTTP version : HTTP/1.1
    • host : -
    • Content-Type : application/json
    • x-eq-ag-timestamp : UNIX timestamp. value MUST NOT differ more than 5 min with timestamp of eqbr’s api-gateway.
    • x-eq-ag-api-key : Api-key issued to projects, used for counting, tracking and logging. The key must be stored in secure place.

    Parameter

    Name Type Required Request Method Description
    microChainId Number True Query Used for specifying user’s network.
    chainNetworkId String True Query Used for specifying user’s network.
    chainId String True Query Used for specifying user’s network.
    functionName String True Query Function name specified in EQ Developer’s Contract Library service.
    contractAddress String True Query Contract-address acquired from deploying contract. Used as an identifier to specify contract in blockchain
    …{optionalData} any Optional Query Required optional data parameters are specified in EQ Developer’s Contract Library service. And may vary for every contract.
  • Response

    key (200)

    Name Type Description
    success Bool ‘true’ returned when all values are present and successfully retrieved its data. ‘false’ returned when address does not match chain’s data.
    data String Optional data parameters are specified in EQ Developer’s Contract Library service. And may vary for every contract.
  • Errors
    • 401 : api-key in the request header not sent or invalid
    • 403 : api-key in the request header has no authority to access the resource
    • 404 : parameter that user has sent is invalid

[Query-2] Chain Query Requests

A. [GET] api/v1/request/nonce

  • Description
    • API for retrieving user’s nonce in certain network.
    • Nonce is the number that is only used once for each address in one network and MUST send correct value while sending out transaction.
    • microChainId, chainNetworkId and chainId are used for specifying user’s network. User’s are able to get detailed information about Network by calling [Network-1-A] API.
  • Request
    • Method : GET
    • Path : api/v1/request/nonce
    • HTTP version : HTTP/1.1
    • host : -
    • Content-Type : application/json
    • x-eq-ag-timestamp : UNIX timestamp. value MUST NOT differ more than 5 min with timestamp of eqbr’s api-gateway.
    • x-eq-ag-api-key : Api-key issued to projects, used for counting, tracking and logging. The key must be stored in secure place.

    Parameter

    Name Type Required Request Method Description
    microChainId Number True Query Used for specifying user’s network.
    chainNetworkId String True Query Used for specifying user’s network.
    chainId String True Query Used for specifying user’s network.
    address String True Query Used for specifying user in blockchain network. Acquired from user’s Private Key
  • Response

    key (200)

    Name Type Description
    success Bool ‘true’ returned when address is correct and successfully retrieved its data. ‘false’ returned when address does not match chain’s requirement
    nonce Number number that is only used once for each address in one network and MUST send correct value while sending out transaction.
  • Errors
    • 401 : api-key in the request header not sent or invalid
    • 403 : api-key in the request header has no authority to access the resource
    • 404 : parameter that user has sent is invalid

[Query-3] Chain Block Query Requests

A. [GET] api/v1/request/blocks

  • Description
    • API for retrieving block data in equilibrium network.
    • API Caller MUST specify minimum block height and maximum block height.
    • API Caller MUST include at least one min and max or height in query parameter.
    • microChainId, chainNetworkId and chainId are used for specifying user’s network. User’s are able to get detailed information about Network by calling [Network-1-A] API.
  • Request
    • Method : GET
    • Path : api/v1/request/blocks
    • HTTP version : HTTP/1.1
    • host : -
    • Content-Type : application/json
    • x-eq-ag-timestamp : UNIX timestamp. value MUST NOT differ more than 5 min with timestamp of eqbr’s api-gateway.
    • x-eq-ag-api-key : Api-key issued to projects, used for counting, tracking and logging. The key must be stored in secure place.

    Parameter

    Name Type Required Request Method Description
    microChainId Number True Query Used for specifying user’s network.
    chainNetworkId String True Query Used for specifying user’s network.
    chainId String True Query Used for specifying user’s network.
    min Number Optional Query Specifies the minimum block height to select from.
    max Number Optional Query Specifies the maximum block height to select to.
    height Number Optional Query Specifies certain block height to retrieve data from.
  • Response

    key (200)

    Name Type Description
    success Bool ‘true’ returned when address is correct and successfully retrieved its data. ‘false’ returned when address does not match chain’s requirement
    blocks Object [] block object

    blocks object array

    Name Type Description
    height Number The block height where user aims to retreive
    round Number round
    block_proposer String Node that proposed the block.
    priority_hash String priority hash
    prev_block_hash String previous block’s hash data
    block_hash String current block’s hash data
    vote_hash String vote hash
    transaction_count Number total number of transactions in block
    timestamp Number time when the block is confirmed
    pub_key String public-key of the node
    signature String signature
    equalizers_list Array array of equalizers that has participated in consensus.
  • Errors
    • 401 : api-key in the request header not sent or invalid
    • 403 : api-key in the request header has no authority to access the resource
    • 404 : parameter that user has sent is invalid

[Query-4] Chain Receipt Query Requests

A. [GET] api/v1/request/receipts

  • Description
    • API for retrieving receipts in equilibrium network.
    • Receipts contain block data, transactions and events.
    • API Caller MUST include at least one min and max or height query parameter.
    • microChainId, chainNetworkId and chainId are used for specifying user’s network. User’s are able to get detailed information about Network by calling [Network-1-A] API.
  • Request
    • Method : GET
    • Path : api/v1/request/receipts
    • HTTP version : HTTP/1.1
    • host : -
    • Content-Type : application/json
    • x-eq-ag-timestamp : UNIX timestamp. value MUST NOT differ more than 5 min with timestamp of eqbr’s api-gateway.
    • x-eq-ag-api-key : Api-key issued to projects, used for counting, tracking and logging. The key must be stored in secure place.

    Parameter

    Name Type Required Request Method Description
    microChainId Number True Query Used for specifying user’s network.
    chainNetworkId String True Query Used for specifying user’s network.
    chainId String True Query Used for specifying user’s network.
    min Number Optional Query Specifies the minimum block height to select from.
    max Number Optional Query Specifies the maximum block height to select to.
    height Number Optional Query The block height where user aims to retreive
    transaction Bool Optional Query Default value is set to ‘true’. Sending ‘false’ will not retreive ‘transaction’ key in the response parameter.
    events Bool Optional Query Default value is set to ‘true’. Sending ‘false’ will not retreive ‘events’ key in the response parameter.
  • Response

    key (200)

    Name Type Description
    success Bool ‘true’ returned when block exists and successfully retrieved its data. ‘false’ returned when block does not exist.
    receipts Object [] receipts object.

    receipt object

    Name Type Description
    block_height Number block height
    tx_index Number index of transaction.
    timestamp Number timestamp when the block is confirmed.
    tx_hash String hashed data of transaction object.
    from String address which has sent out the transaction.
    contract String contract address specifying certain contract.
    result Bool ‘true’ returned when success. ’false’ returned when rejected.
    fee String transaction fee spent when sending out transaction.
    signed_tx Object signed transaction object
    events Object [] events object below.

    signed_tx object

    Name Type Description
    tx_hash String hashed data of transaction object.
    transaction Object transaction object.
    pub_key String public key acquired from private key.
    signature String Signature made from signing tx_hash. tx_hash is made from the transaction object below.

    transaction object

    Name Type Description
    network_id String Used for specifing user’s network.
    chain_id String Used for specifing user’s network.
    from String Address of the user whom sending out transaction
    nonce Number nonce of the user whom sending out transaction. Retrieved from API [Query-1-F]
    contract String Contract-address acquired from deploying contract. Used as an identifier to specify contract in blockchain network.
    function String Function name specified in EQ Developer’s Contract Library service.
    data Object Required optional data parameters are specified in EQ Developer’s Contract Library service. And may vary for every contract.

    events object array

    Name Type Description
    emitter String contract address that emit’s the event.
    id String Function name specified in EQ Developer’s Contract Library service.
    args Object Required optional data parameters are specified in EQ Developer’s Contract Library service. And may vary for every contract.
  • Errors
    • 401 : api-key in the request header not sent or invalid
    • 403 : api-key in the request header has no authority to access the resource
    • 404 : parameter that user has sent is invalid

[Transaction-1] Transaction Requests

  • Description
    • API for sending transaction data in equilibrium network.
    • microChainId, chainNetworkId and chainId are used for specifying user’s network. User’s are able to get detailed information about Network by calling [Network-1-A] API.
  • Request
    • Method : POST
    • Path : api/v1/request/transaction
    • HTTP version : HTTP/1.1
    • host : -
    • Content-Type : application/json
    • x-eq-ag-timestamp : UNIX timestamp. value MUST NOT differ more than 5 min with timestamp of eqbr’s api-gateway.
    • x-eq-ag-api-key : Api-key issued to projects, used for counting, tracking and logging. The key must be stored in secure place.

    Parameter

    Name Type Required Request Method Description
    microChainId Number True Query Used for specifing user’s network.
    chainNetworkId String True Query Used for specifing user’s network.
    chainId String True Query Used for specifing user’s network.
    transaction Object True Body Transaction object specified below.
    pubKey String True Body Required optional data parameters are specified in EQ Developer’s Contract Library service. And may vary for every contract.
    signature String True Body Signature made from signing tx_hash. tx_hash is made from the transaction object below.

    Transaction Object

    Name Type Required Request Method Description
    network_id String True Body Used for specifing user’s network.
    chain_id String True Body Used for specifing user’s network.
    from String True Body Address of the user whom sending out transaction
    nonce Number True Body nonce of the user whom sending out transaction. Retrieved from API [Query-2-A]
    contract String True Body Contract-address acquired from deploying contract. Used as an identifier to specify contract in blockchain network.
    function String True Body Function name specified in EQ Developer’s Contract Library service.
    data Object True Body Required optional data parameters are specified in EQ Developer’s Contract Library service. And may vary for every contract.
  • Response

    key (200)

    Name Type Description
    success Bool ‘true’ returned when transaction has successfully sent to equibrillium network. ’false’ returned when transaction has not been successfully sent to equilibrium network.
    tx_hash String Transaction hash made from transaction object sent in the request body.
  • Errors
    • 401 : api-key in the request header not sent or invalid
    • 403 : api-key in the request header has no authority to access the resource
    • 404 : parameter that user has sent is invalid
