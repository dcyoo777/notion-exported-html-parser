[Query-1] Contract Query Requests

A. [GET] api/v1/request/query

  • Description
    • API for retrieving stored data in equilibrium network.
    • microChainId, chainNetworkId and chainId are used for specifying user’s network. User’s are able to get detailed information about Network by calling [Network-1-A] API.
  • Request
    • Method : GET
    • Path : api/v1/request/query
    • HTTP version : HTTP/1.1
    • host : -
    • Content-Type : application/json
    • x-eq-ag-timestamp : UNIX timestamp. value MUST NOT differ more than 5 min with timestamp of eqbr’s api-gateway.
    • x-eq-ag-api-key : Api-key issued to projects, used for counting, tracking and logging. The key must be stored in secure place.

    Parameter

    NameTypeRequiredRequest MethodDescription
    microChainIdNumberTrueQueryUsed for specifying user’s network.
    chainNetworkIdStringTrueQueryUsed for specifying user’s network.
    chainIdStringTrueQueryUsed for specifying user’s network.
    functionNameStringTrueQueryFunction name specified in EQ Developer’s Contract Library service.
    contractAddressStringTrueQueryContract-address acquired from deploying contract. Used as an identifier to specify contract in blockchain
    …{optionalData}anyOptionalQueryRequired optional data parameters are specified in EQ Developer’s Contract Library service. And may vary for every contract.
  • Response

    key (200)

    NameTypeDescription
    successBool‘true’ returned when all values are present and successfully retrieved its data. ‘false’ returned when address does not match chain’s data.
    dataStringOptional data parameters are specified in EQ Developer’s Contract Library service. And may vary for every contract.
  • Errors
    • 401 : api-key in the request header not sent or invalid
    • 403 : api-key in the request header has no authority to access the resource
    • 404 : parameter that user has sent is invalid

[Query-2] Chain Query Requests

A. [GET] api/v1/request/nonce

  • Description
    • API for retrieving user’s nonce in certain network.
    • Nonce is the number that is only used once for each address in one network and MUST send correct value while sending out transaction.
    • microChainId, chainNetworkId and chainId are used for specifying user’s network. User’s are able to get detailed information about Network by calling [Network-1-A] API.
  • Request
    • Method : GET
    • Path : api/v1/request/nonce
    • HTTP version : HTTP/1.1
    • host : -
    • Content-Type : application/json
    • x-eq-ag-timestamp : UNIX timestamp. value MUST NOT differ more than 5 min with timestamp of eqbr’s api-gateway.
    • x-eq-ag-api-key : Api-key issued to projects, used for counting, tracking and logging. The key must be stored in secure place.

    Parameter

    NameTypeRequiredRequest MethodDescription
    microChainIdNumberTrueQueryUsed for specifying user’s network.
    chainNetworkIdStringTrueQueryUsed for specifying user’s network.
    chainIdStringTrueQueryUsed for specifying user’s network.
    addressStringTrueQueryUsed for specifying user in blockchain network. Acquired from user’s Private Key
  • Response

    key (200)

    NameTypeDescription
    successBool‘true’ returned when address is correct and successfully retrieved its data. ‘false’ returned when address does not match chain’s requirement
    nonceNumbernumber that is only used once for each address in one network and MUST send correct value while sending out transaction.
  • Errors
    • 401 : api-key in the request header not sent or invalid
    • 403 : api-key in the request header has no authority to access the resource
    • 404 : parameter that user has sent is invalid

[Query-3] Chain Block Query Requests

A. [GET] api/v1/request/blocks

  • Description
    • API for retrieving block data in equilibrium network.
    • API Caller MUST specify minimum block height and maximum block height.
    • API Caller MUST include at least one min and max or height in query parameter.
    • microChainId, chainNetworkId and chainId are used for specifying user’s network. User’s are able to get detailed information about Network by calling [Network-1-A] API.
  • Request
    • Method : GET
    • Path : api/v1/request/blocks
    • HTTP version : HTTP/1.1
    • host : -
    • Content-Type : application/json
    • x-eq-ag-timestamp : UNIX timestamp. value MUST NOT differ more than 5 min with timestamp of eqbr’s api-gateway.
    • x-eq-ag-api-key : Api-key issued to projects, used for counting, tracking and logging. The key must be stored in secure place.

    Parameter

    NameTypeRequiredRequest MethodDescription
    microChainIdNumberTrueQueryUsed for specifying user’s network.
    chainNetworkIdStringTrueQueryUsed for specifying user’s network.
    chainIdStringTrueQueryUsed for specifying user’s network.
    minNumberOptionalQuerySpecifies the minimum block height to select from.
    maxNumberOptionalQuerySpecifies the maximum block height to select to.
    heightNumberOptionalQuerySpecifies certain block height to retrieve data from.
  • Response

    key (200)

    NameTypeDescription
    successBool‘true’ returned when address is correct and successfully retrieved its data. ‘false’ returned when address does not match chain’s requirement
    blocksObject []block object

    blocks object array

    NameTypeDescription
    heightNumberThe block height where user aims to retreive
    roundNumberround
    block_proposerStringNode that proposed the block.
    priority_hashStringpriority hash
    prev_block_hashStringprevious block’s hash data
    block_hashStringcurrent block’s hash data
    vote_hashStringvote hash
    transaction_countNumbertotal number of transactions in block
    timestampNumbertime when the block is confirmed
    pub_keyStringpublic-key of the node
    signatureStringsignature
    equalizers_listArrayarray of equalizers that has participated in consensus.
  • Errors
    • 401 : api-key in the request header not sent or invalid
    • 403 : api-key in the request header has no authority to access the resource
    • 404 : parameter that user has sent is invalid

[Query-4] Chain Receipt Query Requests

A. [GET] api/v1/request/receipts

  • Description
    • API for retrieving receipts in equilibrium network.
    • Receipts contain block data, transactions and events.
    • API Caller MUST include at least one min and max or height query parameter.
    • microChainId, chainNetworkId and chainId are used for specifying user’s network. User’s are able to get detailed information about Network by calling [Network-1-A] API.
  • Request
    • Method : GET
    • Path : api/v1/request/receipts
    • HTTP version : HTTP/1.1
    • host : -
    • Content-Type : application/json
    • x-eq-ag-timestamp : UNIX timestamp. value MUST NOT differ more than 5 min with timestamp of eqbr’s api-gateway.
    • x-eq-ag-api-key : Api-key issued to projects, used for counting, tracking and logging. The key must be stored in secure place.

    Parameter

    NameTypeRequiredRequest MethodDescription
    microChainIdNumberTrueQueryUsed for specifying user’s network.
    chainNetworkIdStringTrueQueryUsed for specifying user’s network.
    chainIdStringTrueQueryUsed for specifying user’s network.
    minNumberOptionalQuerySpecifies the minimum block height to select from.
    maxNumberOptionalQuerySpecifies the maximum block height to select to.
    heightNumberOptionalQueryThe block height where user aims to retreive
    transactionBoolOptionalQueryDefault value is set to ‘true’. Sending ‘false’ will not retreive ‘transaction’ key in the response parameter.
    eventsBoolOptionalQueryDefault value is set to ‘true’. Sending ‘false’ will not retreive ‘events’ key in the response parameter.
  • Response

    key (200)

    NameTypeDescription
    successBool‘true’ returned when block exists and successfully retrieved its data. ‘false’ returned when block does not exist.
    receiptsObject []receipts object.

    fromStringaddress which has sent out the transaction.
    contractStringcontract address specifying certain contract.
    resultBool‘true’ returned when success. ’false’ returned when rejected.
    feeStringtransaction fee spent when sending out transaction.
    signed_txObjectsigned transaction object
    eventsObject []events object below.

    signed_tx object

    NameTypeDescription
    tx_hashStringhashed data of transaction object.
    transactionObjecttransaction object.
    pub_keyStringpublic key acquired from private key.
    signatureStringSignature made from signing tx_hash. tx_hash is made from the transaction object below.

    transaction object

    NameTypeDescription
    network_idStringUsed for specifing user’s network.

    fromStringAddress of the user whom sending out transaction
    nonceNumbernonce of the user whom sending out transaction. Retrieved from API [Query-1-F]
    contractStringContract-address acquired from deploying contract. Used as an identifier to specify contract in blockchain network.

    • 404 : parameter that user has sent is invalid
